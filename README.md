# Imitation Learning for GUI Navigation (MiniWoB++)

Train an agent to navigate web/GUI tasks **from pixels** via **behavioral
cloning (BC)**. The agent observes a screenshot and predicts *where to click* to
make progress on the task — learning purely from demonstration frames, never the
DOM.

- **Environment:** [MiniWoB++](https://github.com/Farama-Foundation/miniwob-plusplus)
  (Farama `miniwob`), a suite of small web tasks rendered in Chrome via Selenium.
- **Demonstrations:** generated by *scripted oracle experts* that read the DOM to
  click the correct target. Only successful episodes are kept, yielding clean
  `(screenshot, click)` data. Each episode is a short "demonstration video".
- **Method:** behavioral cloning. A CNN maps a screenshot → a click-location
  heatmap (spatial cross-entropy over a grid). Inference clicks the argmax cell.

## Pipeline

```
collect.py   oracle experts -> data/demos/<task>.npz   (screenshot, click) pairs
dataset.py   load demos, pixel click -> grid cell
model.py     CNN encoder -> grid*grid click logits
train.py     spatial cross-entropy BC, saves runs/bc/best.pt
evaluate.py  run policy live in the browser, optional random-click baseline
```

## Setup

Requires Google Chrome (Selenium drives it). On macOS:
`brew install --cask google-chrome`.

```bash
uv sync
```

## Run

```bash
# 1. Collect demonstrations (writes data/demos/*.npz)
uv run python -m il.collect

# 2. Train the BC click policy (writes runs/bc/best.pt)
uv run python -m il.train

# 3. Evaluate live in the browser
uv run python -m il.evaluate
```

Configuration lives in `configs/default.yaml` (tasks, episode counts, model
size, training hyperparameters). Override the task list or episode counts on the
command line, e.g. `... -m il.collect --tasks click-button --episodes-per-task 50`.

## Tasks

The default config targets four single-click MiniWoB++ tasks the oracle solves
reliably: `click-test`, `click-test-2`, `click-button`, `click-link`.

## Extending

- **More tasks:** add to `tasks:` in the config. Multi-step or typing tasks need
  a richer action head (e.g. an action-type classifier + text head) and an oracle
  that emits those actions — the framework already records `action_type`.
- **Stronger encoder:** swap the CNN in `model.py` for a pretrained ResNet/ViT.
- **Distribution shift:** BC is open-loop; consider DAgger (re-query the oracle on
  states the policy visits) or GAIL for robustness.

## Drawing Imitation Demo

This repo also includes a tiny imitation-learning demo for drawing from example
GIFs. It infers expert brush positions from newly appearing ink between frames,
then trains a policy to draw the target image from a blank canvas.

```bash
# Create a few synthetic drawing "video" GIFs.
uv run python -m drawing_il.make_demos

# Train from those GIFs.
uv run python -m drawing_il.train

# Roll out the learned drawing policy into a GIF.
uv run python -m drawing_il.rollout --target data/drawing/videos/circle.gif
```

You can replace `data/drawing/videos/*.gif` with your own simple drawing GIFs
or folders of frame images. For MP4 videos, first export frames/GIFs with a
video tool, then point `drawing_il.train --data-dir` at them.

## WebLINX Action Imitation

WebLINX is a real-world conversational web-navigation dataset with about 100K
interactions across 2300 expert demonstrations. This repo includes a lightweight
first baseline that trains a prompt-to-action-string model from Hugging Face
WebLINX records.

```bash
# Inspect raw WebLINX records and fields.
uv run python -m weblinx_il.inspect --split validation --n 2

# Train a small prompt -> action string baseline on a subset.
uv run python -m weblinx_il.train --train-limit 2000 --val-limit 300

# Fine-tune GPT-2 on WebLINX action strings.
uv run python -m weblinx_il.train_gpt2 \
  --train-limit 2000 \
  --val-limit 300 \
  --epochs 1 \
  --batch-size 2 \
  --grad-accum 8

# Try one prompt after training.
uv run python -m weblinx_il.predict_gpt2 \
  --checkpoint runs/weblinx/gpt2 \
  --prompt "Instructor: open the shopping cart. Current page: product page."

# Evaluate the GPT-2 checkpoint on held-out WebLINX records.
uv run python -m weblinx_il.evaluate \
  --checkpoint runs/weblinx/gpt2 \
  --split validation \
  --limit 300

# Watch the GPT-2 action model control a visible Chrome browser.
uv run python -m weblinx_il.live_browser \
  --checkpoint runs/weblinx/gpt2 \
  --url "https://www.encyclopedia.com/" \
  --instruction "Search for biotechnology"
```

This first WebLINX path is text/action-only. The GPT-2 trainer uses the WebLINX
state/history/candidates prompt and fine-tunes GPT-2 to complete the next action
string. The live browser controller opens Chrome, extracts visible DOM
candidates, predicts a WebLINX-style action, and executes supported actions such
as `click`, `text_input`, and `load`.

## Layout

```
configs/default.yaml   all hyperparameters
src/il/
  env.py        MiniWoB env wrapper (click action space, screenshots)
  experts.py    scripted oracle target selection
  collect.py    demonstration collection
  dataset.py    torch Dataset, pixel<->grid-cell conversion
  model.py      CNN click-heatmap policy
  train.py      BC training loop
  evaluate.py   live browser evaluation
  utils.py      config / device / seeding
data/demos/     collected .npz demonstrations (gitignored)
runs/bc/        checkpoints (gitignored)
```
